# Project 01 — Azure Cost Governance & Portfolio Foundation

## Project Overview

This project establishes the foundation for a safe and organized Azure lab environment.

Before deploying Azure resources, it is important to define basic governance standards such as naming conventions, cost controls, documentation structure, and resource organization.

This project is part of my Azure Hybrid Identity & Cloud Systems Engineering Portfolio.

## Business Problem

Cloud environments can become expensive, disorganized, and difficult to manage when resources are created without standards.

Common problems include:

- Unclear resource names
- No monthly budget
- No ownership tracking
- No cleanup process
- Inconsistent documentation
- Difficulty troubleshooting or auditing resources

A cloud engineer should understand how to create structure before building infrastructure.

## Project Objective

The objective of this project is to create a repeatable foundation for future Azure portfolio labs.

This project will document:

- Azure resource naming standards
- Cost governance planning
- Screenshot documentation standards
- Portfolio folder structure
- Future tagging standards
- Future budget and alert configuration
